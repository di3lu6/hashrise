Prueba de penetración Interna

Descripción

Es una prueba de seguridad que simula un ataque real desde la red interna LAN (por un intruso o empleado disgustado), con el objetivo de conocer cuan vulnerable es su organización realmente.

La prueba de penetración comienza con las mismas tareas que el análisis de vulnerabilidades. Luego de ello, el riesgo real es verificado aprovechando las vulnerabilidades y todas las posibles combinaciones.

Las fases del servicio son:

  • Descubrimientos.
  • Análisis de vulnerabilidades.
  • Explotación.
  • Escalación de privilegios y mantenimiento de accesos

Pruebas de penetración WiFi

Pruebas de penetración WiFi

Descripción

Las redes inalámbricas son una extensión del perímetro de su empresa, y se pueden utilizar como una puerta de entrada a su infraestructura interna.

Un punto de acceso no autorizado, instalado por un empleado, que no sigue las directrices de seguridad de la empresa se puede utilizar para poner en peligro su seguridad.

Este tipo de prueba incluye un análisis de espectro Wi-Fi para detectar puntos de acceso, y un análisis de la seguridad de las redes inalámbricas de la compañía con el fin de evaluar las medidas de seguridad implementadas y detectar posibles vulnerabilidades.

Pruebas de Ingeniería Social

Pruebas de Ingeniería Social

Descripción

La ingeniería social utiliza la manipulación, la influencia y el engaño para conseguir de una persona información privilegiada de confianza dentro de una organización, para cumplir con una petición, o divulgar información que beneficie al atacante.

Mediante el uso de técnicas como "spear phishing" y llamadas telefónicas falsas a los empleados de su empresa, tratamos de conseguir el acceso a la información o los sistemas sensibles. De esta manera, se llega a conocer el nivel real de conocimiento de su personal.

Application Security
Análisis de aplicaciones Web

Análisis de aplicaciones Web

Descripción

Durante el análisis de una aplicación web, las pruebas manuales y automatizadas se ejecutan con el fin de detectar las implementaciones inseguras o vulnerabilidades que podrían haber sido introducidas por errores de desarrollo. Estas vulnerabilidades suelen permitir el acceso a la información sensible, escalando los privilegio dentro de la aplicación, e incluso accediendo y modificando el funcionamiento interno del servidor.

El proceso analítico utilizado por Security Advisor se basan en las metodologías de OWASP y OSSTMM, y siempre incluye controles para las vulnerabilidades que componen la lista Top 10 de OWASP.

Debido a que la lógica de cada aplicación es diferente y depende enormemente de las reglas del negocio, también buscamos vulnerabilidades lógicas que no serían detectadas mediante el análisis automático.

Auditoría de Código Fuente

Auditoría de Código Fuente

Descripción

La auditoría al código fuente (o de auditoría "caja blanca") le ayuda a obtener una mejor comprensión del nivel de seguridad de las aplicaciones desarrolladas, mediante la detección de vulnerabilidades y problemas de seguridad introducidas durante las etapas de "análisis de requisitos", "Diseño" y "Desarrollo" . Debido a que el código fuente está disponible para el auditor, es posible detectar vulnerabilidades que no serían detectados con otros abordajes.

Cada componente de seguridad de las aplicaciones está profundamente analizada con el apoyo del equipo de desarrollo. Esto nos permite obtener un mejor conocimiento de la lógica de la aplicación y del negocio.

Probamos con especial atención los componentes críticos como:

  • Autenticación.
  • Autorización.
  • Validación de datos de entrada.
  • Error y el manejo de excepciones.
  • Llamadas al sistema operativo.
PCI-DSS Required Services
Pruebas de penetración (PCI)

Pruebas de penetración (PCI)

Descripción

El PCI-DSS (PCI DSS) requiere pruebas de penetración que se lleven a cabo para garantizar la seguridad del entorno de datos de tarjetas de crédito. Desde PCI-DSS 3.0 todos los comerciantes, procesadores, pasarelas de pago y proveedores de servicios deben llevar a cabo tareas de pruebas de penetración para:

  • Cubrir todo el entorno de datos de titulares de tarjetas.
  • Cubrir tanto la capa de aplicación y las amenazas de capa de red.
  • Se basan en un modelo aceptado por la industria.
  • Cubrir las amenazas tanto externas e internas.

Las pruebas de penetración deben hacerse sobre una base anual y después de cualquier cambio significativo en la infraestructura o aplicaciones.

Servicios de pruebas de penetración de Hashrise se alinean para el cumplimiento de PCI-DSS.

Curso OWASP Top 10

Curso OWASP Top 10

Description

El curso tiene como objetivo mostrar y explicar la lista de OWASP Top 10, la misma existe tanto para aplicaciones web, móviles y cloud. Contienen las 10 vulnerabilidades más críticas que por lo general se encuentran en análisis de aplicaciones.

Durante este entrenamiento, los siguientes temas serán cubiertos:

  • Explicación de todas las vulnerabilidades en la lista Top 10.
  • Diferentes formas de detectarlas (manual y automática).
  • Demostraciones prácticas de su explotación.
  • Mejores prácticas de desarrollo para evitar cada una de las vulnerabilidades.

Duración: 12hs (1 día y medio, o 3 días y medio)
