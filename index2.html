Penetration Testing
Internal Penetration Testing

Internal Penetration Testing

Description

It is a security test where we simulate a real attack from within the internal network (by an intruder or a disgrunted employee), in order to detect how vulnerable your company's network really is.

The Penetration Testing starts with the same tasks included in a Vulnerability Assessment. After that, the real risk level is verified by exploiting the found vulnerabilities and all their possible combinations.

Its phases are:

  • Discovery.
  • Vulnerability Analysis.
  • Exploitation.
  • Privilege escalation and mantaining access.

Wireless Penetration Testing

Wireless Penetration Testing

Description

Wireless netwoks are an extension of your company's perimeter, and can be used as a gateway to your internal infrastructure.

An unauthorized Access Point, installed by an employee, which doesn't follow the company's security guidelines can be used to compromise your security.

This type of test includes a WiFi spectrum analysis to detect rogue Access Points, and a security analysis of the company's wireless networks in order to assess the implemented security measures and detect possible vulnerabilities.

RFID/NFC Access Testing

RFID/NFC Access Testing

Description

Each day, more companies use RFID/NFC technology for employee identification, and to control access to areas and services.

A vulnerable implementation of this kind of technology could allow an attacker to gain physical access to your company, impersonate valid employees and other sorts of attacks.

In this type of testing, we try to bypass the access controls by using specialized tools and hardware, so that we can assess the implemented security measures and possible existing vulnerabilities.

Application Security
Análisis de aplicaciones Web

Análisis de aplicaciones Web

Descripción

Durante el análisis de una aplicación web, las pruebas manuales y automatizadas se ejecutan con el fin de detectar las implementaciones inseguras o vulnerabilidades que podrían haber sido introducidas por errores de desarrollo. Estas vulnerabilidades suelen permitir el acceso a la información sensible, escalando los privilegio dentro de la aplicación, e incluso de funcionamiento del sistema de acceso.

El proceso analítico utilizado por Hashrise se basan en las metodologías de OWASP y OSSTMM, y siempre incluye controles para las vulnerabilidades que componen la lista Top 10 de OWASP.

Debido a que la lógica de cada aplicación es diferente y depende enormemente de las reglas del bussiness, también buscamos vulnerabilidades lógicas bussiness que no serían detectados mediante el análisis automático.

Auditoría de Código Fuente

Descripción

  • Autorización.

  • Sistema operativo llama.
PCI-DSS Required Services
Pruebas de penetración (PCI)

Pruebas de penetración (PCI)

Descripción

El PCI-DSS (PCI DSS) requiere pruebas de penetración que se lleven a cabo para garantizar la seguridad del entorno de datos de tarjetas de crédito. Desde PCI-DSS 3.0 todos los comerciantes, procesadores, pasarelas de pago y proveedores de servicios deben llevar a cabo tareas de pruebas de penetración para:

  • Cubrir todo el entorno de datos de titulares de tarjetas.
  • Cubrir tanto la capa de aplicación y las amenazas de capa de red.
  • Se basan en un modelo aceptado por la industria.
  • Cubrir las amenazas tanto externas e internas.

Las pruebas de penetración deben hacerse sobre una base anual y después de cualquier cambio significativo en la infraestructura o aplicaciones.

Servicios de pruebas de penetración de Hashrise se alinean para el cumplimiento de PCI-DSS.

Training/Courses
IoT Training

IoT Training

Description

This workshop will provide you with all the tools, materials, and references for further study and research. The workshop covers from the Low Frequency band (mainly used for individual physical access to buildings, garages, hotels, etc.) to the High Frequency band, where NFC is the main term we are going to discuss about.

We will understand which type of access cards can be emulated or even cloned; then we are going to use traditional NFC USB readers, we will compile and execute the famous LibNFC, and play with some special hardware like the Proxmark3 and HydraNFC.

Arduino examples will be shown to deal with both Low and High frequency cards and tokens. We will try to create our own cloner in less than 50 lines of code!

The last part of the workshop will review HID iClass vulnerabilities and focus on NXP Mifare Classic technology and public hacks.

By the end, we will discuss some case studies, practical hacks and lessons learned from working systems the can be found in the wild.


Duration: 8hs (1 full day or 2 half days)
